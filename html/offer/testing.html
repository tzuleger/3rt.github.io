Testing

We develop, we maintain, but most importantly, we test.

Through the development process, our developers thoroughly walk through the code they write. With the layers of review, we also assign testers to our developing applications. Our testers walk through the code hundreds of times as if they are actual users looking for any deficienies that may have been missed.

Manual Testing

Automated Testing

Manually testing can leave room for human error or concurrency errors. That is why we use testing libraries like the Selenium Webdriver so we can guarantee that all corner cases of your web application have been tested. We also use SpecFlow for human-readable test cases to provide as much transparency between the test and the customer. The goal of our automated testing is to simulate an actual work environment and catch bugs that a single user cannot catch.

Unit Testing is one of the most vital parts to maintaining code and applications. Our developers aim for a 90% or better code coverage for all unit tests. Knowing that the code has been run in a unit test ensures that the code is behaving as expected.
